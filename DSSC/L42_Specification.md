## 第1章：問題の定義

### 1.0 「最良の分割」の定義（確定事項と未確定事項の区別）

#### 確定していること（L38〜L41で実証済み）

```
【確定①】margin_pred を最大化する分割 = DSSCが最も余裕をもって解ける分割
  根拠：L41で margin_pred の誤差16.4%以内の予測精度を実証済み
  意味：margin_pred が最大の分割を選べば、DSSCが正解を返す確率が最も高い

【確定②】margin_pred = C(W_intra, W_inter) × (1 - λ2/λ1)²
  根拠：L41設計マップ（9点実測）
  意味：分割を決めれば、シミュレーションなしで margin_pred を計算できる

【確定③】W_intra↑・W_inter↓ の方向が良い分割
  根拠：L41単調性チェック（全行・全列で単調性確認）
  意味：「結合の強いノード同士をペアにする」が正しい方向性
```

#### L42で確認すること（未確定）

```
【未確定A】margin_pred 最大化 ＝ 問題の正答率最大化、は同値か？
  現状：margin_pred > 0 → DETECTED は確認済み
  未確定：margin_pred が大きいほど正答率が高いかどうか
  → L42のシミュレーション結果で検証する

【未確定B】「解きやすさ」はmarginだけで決まるか？
  margin は「第2層を検出できるかどうか」の指標
  ≠ 「元の問題（タンパク質折り畳み等）の最適解を
      正しく求められるかどうか」

  具体的なギャップ：
    ・問題をDSSCに変換する際の「情報の損失」
      （不均一J行列を均一W_intra・W_interで近似する誤差）
    ・変換後の問題が「元の問題の正解」と
      対応しているかどうか
  → これはL43〜L44の課題

【未確定C】貪欲法でどの程度の近似精度が出るか？
  直感：「強い結合をペア内に集める」= 最適に近いはず
  未確定：実際に何%の精度で最適解に近づくか
  → L42の完全列挙（Algo3）との比較で確認する

【L42の「最良」の操作的定義】
  L42では以下を「最良」とする：

  「margin_pred を最大化する分割」

  これは：
    ○ DSSCが解けるかどうかの予測精度を最大化する
    ○ シミュレーション前に計算コストゼロで評価できる
  これではない：
    × 元の問題（タンパク質構造等）の正答率を保証する
    × 情報損失ゼロの変換を保証する

  「margin_pred最大化」から「元の問題の正答率最大化」への
  接続はL43以降の課題として明示的に分離する
```

### 1.1 グラフ分割問題とは

```
入力：
  N個のノード（偶数）と
  ノード間の結合強度 J[i,j]（任意、不均一でよい）

  例（8残基ペプチドの場合）：
    ノード = アミノ酸残基 {A,B,C,D,E,F,G,H}
    J[i,j] = 残基間の相互作用エネルギー

出力：
  N/2 個のペア（複合体）の組み合わせ
  例：{A,C}, {B,F}, {D,H}, {E,G}

目標：
  分割後の W_intra（ペア内平均結合）を最大化
  分割後の W_inter（ペア間平均結合）を最小化
  → 設計マップ上で C が最大になる分割を選ぶ

制約：
  各ノードは必ず1つのペアに属する
  （完全マッチング条件）
```

### 1.2 なぜ分割が重要か

```
同じJ行列でも分割次第でmarginが大きく変わる

悪い分割の例（結合の弱いノード同士をペアに）：
  W_intra = 0.1, W_inter = 0.5
  → 設計マップ: C ≈ 0.03（非常に小さい）
  → margin ≈ 0.03 × (1-0.83)² ≈ 0.001（ギリギリ）

良い分割の例（結合の強いノード同士をペアに）：
  W_intra = 0.8, W_inter = 0.1
  → 設計マップ: C ≈ 0.8
  → margin ≈ 0.8 × (1-0.5)² ≈ 0.2（余裕あり）

= 同じ問題でも分割の良し悪しで margin が100倍以上変わりうる
```

### 1.3 完全列挙の限界

```
N=8 の場合：
  可能なペア分割数 = (8-1)!! = 7×5×3×1 = 105通り
  → 完全列挙は可能

N=12 の場合：
  可能なペア分割数 = (12-1)!! = 10395通り
  → 完全列挙は可能だが重い

N=20 の場合：
  可能なペア分割数 = (20-1)!! ≈ 6.5×10⁸通り
  → 完全列挙は不可能

→ N=8では完全列挙で最適解を確認
  N≥12では効率的なアルゴリズムが必要
```

## 第2章：アルゴリズム設計

### 2.1 提案する3つのアルゴリズム

```
Algo1【貪欲法（Greedy）】:
  J[i,j] が大きい順にペアを確定していく
  
  手順：
    1. 全ペア候補をJ[i,j]の降順に並べる
    2. 上から順に「まだペアになっていない」ノード同士を選ぶ
    3. N/2ペアが揃うまで繰り返す
  
  計算量：O(N²log N)
  特徴：高速・実装簡単・最適解の保証なし

Algo2【スペクトル法（Spectral）】:
  J行列の固有ベクトルを使って分割する
  
  手順：
    1. J行列の固有値分解
    2. 第1固有ベクトルの成分が正のノードと負のノードをペアにする
       （物理的意味：最もエネルギーが下がる分割）
    3. 残ったノードを再帰的に処理
  
  計算量：O(N³)
  特徴：物理的根拠あり・局所最適に陥る可能性

Algo3【完全列挙（Exhaustive）】:
  全ての分割を試してC_predが最大のものを選ぶ
  
  手順：
    1. (N-1)!!通りの完全マッチングを列挙
    2. 各分割に対して W_intra, W_inter を計算
    3. 設計マップで C_pred を計算
    4. C_pred が最大の分割を返す
  
  計算量：O((N-1)!! × N²)
  特徴：最適解保証・N=8のみ実用的
  役割：Algo1・Algo2の精度評価基準
```

### 2.2 評価指標

```
各アルゴリズムの出力を以下で評価：

① C_pred（予測C値）：設計マップから計算
    高いほど良い

② margin_pred（予測margin）：C × (1-r)²
    高いほど良い

③ margin_actual（実測margin）：DSSCシミュレーション
    予測との誤差も確認

④ 最適解との比率：
    score = Algo1のmargin_pred / Algo3のmargin_pred
    → 1.0に近いほど最適解に近い
```

## 第3章：実験設計

### 3.1 テストJ行列の種類

```
テストケースA【均一J行列】：
  J_intra = 1.0（全ペア同一）
  J_inter = 0.10（全ペア間同一）
  → L41の実績点（C=0.364）と一致するはず
  → アルゴリズムの動作確認用

テストケースB【不均一J行列（弱）】：
  J[i,j] = 一様分布 U(0.05, 0.20) でランダム生成
  → 不均一だがL41の設計マップ範囲内
  → アルゴリズムの有効性検証

テストケースB' 【不均一J行列（強）】：
  J[i,j] = 一様分布 U(0.01, 1.00) でランダム生成
  → 強く不均一なJ行列
  → 設計ツールの適用限界を探る

テストケースC【タンパク質模擬J行列】：
  実際のアミノ酸相互作用ポテンシャルに基づく
  接触ペアのみ J[i,j] ≠ 0（疎な行列）
  → タンパク質計算への適用可能性の第一検証
  ※ 最初は4残基程度の極小ペプチドから始める
```

### 3.2 実験フロー

```
Step1：各テストケースのJ行列を生成

Step2：3アルゴリズムで分割を実行
  → 各分割の W_intra, W_inter, C_pred, margin_pred を記録

Step3：DSSCシミュレーション（上位3分割のみ）
  → margin_actual を測定
  → 予測誤差を確認

Step4：アルゴリズム比較
  → 最適解（Algo3）に対するAlgo1・Algo2のスコアを計算
  → 計算時間も記録

Step5：設計指針の更新
  → どのアルゴリズムがどの状況で有効かを整理
```

### 3.3 実験パターン一覧

| No | テストケース | アルゴリズム | シミュレーション | 目的 |
|----|------------|------------|--------------|------|
| T1 | A（均一） | Algo1,2,3 | 最良1点 | 動作確認 |
| T2 | B（不均一弱）×5試行 | Algo1,2,3 | 最良各1点 | 有効性検証 |
| T3 | B'（不均一強）×5試行 | Algo1,2,3 | 最良各1点 | 限界探索 |

## 第4章：予想される結果

### 4.1 楽観的シナリオ（期待値）

```
Algo1（貪欲法）の予測スコア：
  均一J行列（T1）  : 1.00（全分割が等価なので最適）
  不均一弱（T2）   : 0.85〜0.95
  不均一強（T3）   : 0.70〜0.90
  タンパク質模擬（T4）: 0.80〜0.95

→ 貪欲法が「J値の大きいものをペアにする」という
  直感的な方針で80〜95%の精度を達成することが期待される

理由：
  設計マップで C は J_intra の単調増加関数
  → 大きな J[i,j] をペア内に集めることが最適に近い
  → 貪欲法はまさにそれをやっている
```

### 4.2 悲観的シナリオ（課題が残る場合）

```
Algo1のスコアが0.70を下回った場合：
  → 局所的に大きなJ[i,j]をペアにすることが
    グローバル最適ではないケースが多い
  → より高度なアルゴリズム（ハンガリアン法など）が必要

不均一強（T3）でmargin_pred < 0 が多発した場合：
  → 設計マップの「均一近似」が不均一J行列に対して
    精度が出ない
  → 不均一J行列用の補正式が必要（L43課題）

タンパク質模擬（T4）でFAILした場合：
  → 接触行列（疎なJ行列）は均一近似が成立しない
  → 疎行列専用のアプローチが必要
```

### 4.3 どちらのシナリオでも得られるもの

```
「成功」の場合：
  → 任意J行列 → 最適分割 → margin予測という
    パイプラインの第一実証
  → タンパク質計算への橋渡しが見えてくる

「課題が残る」場合：
  → 設計ツールの適用限界が数値で明確になる
  → L43の課題設定に直結する
  → 「できないとわかった」も成果（PMNT_DSSC_Findings.md参照）

どちらでも：
  → 最適分割アルゴリズムという道具が手に入る
  → N=8でのグラフ分割問題が完全に解決される
```

## 第5章：成功の定義

```
完全成功：
  ① T1（均一J行列）でAlgo3と同一の分割を全アルゴリズムが選ぶ
  ② T2（不均一弱）でAlgo1のスコア ≥ 0.85
  ③ T4（タンパク質模擬）でmargin_pred > 0
  ④ 実測誤差 < 20%（L41基準を継承）

部分成功：

  → タンパク質への道はL43以降に先送り

失敗：
  ② が未達（Algo1スコア < 0.85）
  → アルゴリズムの再設計が必要
```

## 付録B：タンパク質模擬J行列の生成方法

```
4残基ペプチド（N=8スピン、各残基=2スピン）の例：

残基間接触エネルギー（MJ行列の簡略版）：
  親水性-親水性 : J = -0.1〜-0.3（反発）
  疎水性-疎水性 : J = +0.3〜+0.8（引力）
  親水性-疎水性 : J = -0.05〜+0.05（弱い相互作用）

接触条件：
  隣接残基（i, i+1）は常に接触
  非隣接残基（|i-j| ≥ 2）は接触確率50%

→ 疎水性コア形成の模擬：
  疎水性残基同士が高いJ値でペアになれば
  W_intraが大きくなり、DSSCで検出しやすくなる
  = 疎水性コア形成問題をDSSCで解く最初のステップ
```
