# L42 最小実装メモ

## 何をするコードか
この最小実装は、DSSC を回す前に

- 意味保存性 `M(P,S)`
- 分離保存性 `D(P,S)`
- 階層保存性 `H(P,S)`
- 追跡可能性 `R(P,S)`

を計算し、**P→S が意味保存写像かどうか** をざっくり判定する。

## 含まれるファイル
- `l42_semantic_model.py`
- `l42_run_example.py`

## 実行方法
```bash
python l42_run_example.py
```

## 想定される計算時間
この最小実装は前段評価器であり、DSSC 本体を回さない。

### ローカルPC / CPU
- N=6〜20 程度の小規模例
- 実行時間: **1秒未満〜数秒**

### 写像候補を多数比較する場合
- 3写像 × 10条件程度
- 実行時間: **数秒〜数十秒**

## さくら高火力は必要か
**不要。**

理由:
- L42 の最初の目的は `P→S` の妥当性検証
- ここでは hyperedge 展開と簡易 pairwise 重み計算が中心
- 行列サイズ 512 や 8192 の時間発展はまだしない

### さくら高火力が必要になる段階
- L42 後半で DSSC 本体まで接続して大量比較するとき
- L43 で S→G→J 比較を多数回し、さらに DSSC まで流すとき
- N=12 以上で本格的に margin を再評価するとき

## 実務推奨
1. まずローカルで `P→S` の評価器を固める
2. 次に S→G の射影比較をローカルで行う
3. その後に G→J→DSSC を高火力でまとめて回す

## 一言
**L42 前半は CPU ローカルで十分。高火力はまだ早い。**
