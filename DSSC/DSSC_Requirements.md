## 第1章：確定済みの物理基盤
### （変更不可・全章の前提）

これ以降の全ての議論の土台。変更は不可。

### 1.1 動作パラメータ（L33確定）

| パラメータ | 値 | 確定方法 |
|-----------|-----|---------|
| g | 2,000 MHz | ハードウェア固定 |
| Δ | 13,000 MHz | L31で確定 |
| J_eff | 308 MHz | g²/Δから導出 |
| κ_eff | 8.85 MHz | κ(g/Δ)²から導出 |
| s* | 0.890 | L33で確定（N非依存） |
| Γ* | 330 MHz | L33で確定 |
| 動作範囲 | s = 0.80〜0.970 | L38 Phase1で確定 |

### 1.2 動作範囲の確定（L38 Phase1確定）

```
sigma✓（信頼できる）：s ≦ 0.970（Γ ≧ 90MHz）
sigma△（信頼できない）：s ≧ 0.985（Γ ≦ 45MHz）

= Γが小さすぎると定常状態が不安定になる
= これはDSSCの物理的制約であり変更不可
```

### 1.3 計算原理（変更不可）

```
公理：物理系はエネルギー最小状態に収束する

H_eff = -J_eff Σ J_{ij}(σ_+σ_- + h.c.) - Γ Σ σ_x

DSSCはJ_{ij}行列の第1固有モードを
s*=0.890で物理的に抽出する機械
```

## 第2章：固有値比の定量的理解
### （第3章以降の判断基準）

第1章の物理から導出される。第3章の要件定義の根拠となる。

### 2.1 marginと固有値比の関係式（L35〜L37実証）

```
margin ≈ C × (1 - λ2/λ1)^2

実測値：
  λ2/λ1=0.667（N=4）: margin=+0.0418
  λ2/λ1=0.571（N=6）: margin=+0.0786
  λ2/λ1=0.500（N=8）: margin=+0.0911

フィット：C ≈ 0.38（N=4,6,8で一定）
指数α = 1.97 ≈ 2.0

物理的根拠：
  α=2は量子力学の2次摂動論から導出される
  = H_Γが固有モードを混合する効果 ∝ (行列要素)^2
```

### 2.2 固有ベクトルの構造とN依存性（解析的導出）

```
第1固有ベクトル|v1>：
  全スピン均等：|v1_i|^2 = 1/N
  = 全複合体の同相振動

第2固有ベクトル|v2>：
  複合体間の逆相振動
  = DSSCが検出したい構造

空間的重なり（干渉の大きさ）：
  Σ_i |v1_i|^2 |v2_i|^2 = 1/N

  証明：
  = (1/N) × Σ_i |v2_i|^2
  = (1/N) × 1
  = 1/N

→ Nが増えると干渉が減る
→ marginが向上する（スケーリング則の根拠）
```

### 2.3 λ2/λ1のN依存性（4複合体配置）

```
近似式（J_intra=1.0, J_inter=J）：

  λ1 ≈ 1 + (N-2) × J
  λ2 ≈ (N-2) × J

  λ2/λ1 ≈ (N-2)×J / (1 + (N-2)×J)

実測との一致（傾向）：
  N=4: λ2/λ1=0.667（Nが小さいと比率大）
  N=8: λ2/λ1=0.500（Nが大きいと比率小）
  N=∞: λ2/λ1→1（縮退に近づく）

検出可能なJ_interの上限：
  J_inter_max = ε_c / ((N-2)(1-ε_c))

  N=4: J_max≈0.67, N=6: J_max≈0.33
  N=8: J_max≈0.22, N=10: J_max≈0.17
```

### 2.4 配置依存性（L38 Phase1で発見）

```
4複合体配置（ScenarioA型）：
  |v1>と|v2>が空間的に分離
  → margin ∝ (1-λ2/λ1)^2 成立
  → 理論上ε_c ≈ 1（縮退以外は解ける）

階層配置（L37_C型）：
  |v1>と|v2>が同じスピンを共有
  → 干渉が大きい
  → λ2/λ1=0.800で margin=-0.125（検出不可）
  → ε_c(階層) ≈ 0.57〜0.80

重要な発見：
  ε_cは普遍定数ではなく配置依存
  J_{ij}の値だけでなく
  固有ベクトルの構造が解けるかどうかを決める
```

## 第3章：「解ける問題」の要件定義
### （第2章の根拠に基づく定義）

ここから「要件」として定義する。

### 3.1 基本要件（L35〜L37確定）

```
DSSCが解ける問題の必要条件：

要件1（非縮退）：
  J_{ij}行列の最大固有値が一意
  λ1 > λ2（縮退なし）

要件2（符号）：
  全てのJ_{ij} > 0
  （負のJ_{ij}は現在対応不可）

要件3（動作範囲）：
  J_eff × λ1 × t* < π/4
  （sin²の動作範囲内）

要件4（固有値比）：← 今回新たに定義
  λ2/λ1 < ε_c
  （ε_cはL38 Phase2で確定予定）
```

### 3.2 ε_cの現在の推定範囲

```
確定済みデータ：
  λ2/λ1=0.571（N=6）：margin=+0.079 → 検出可 ✓
  λ2/λ1=0.800（N=8 階層）：margin=-0.125 → 検出不可 ✗

現在の推定：
  0.571 ≦ ε_c < 0.800

ただし配置依存のため：
  4複合体配置：ε_c(A) ≈ 1
  階層配置：ε_c(C) ≈ 0.571〜0.800

→ L38 Phase2で確定する（第4章）
```

### 3.3 要件4の配置設計への翻訳

```
「λ2/λ1 < ε_c」を実現するための
J_{ij}設計ガイドライン：

① J行列を設計する
② numpy等で固有値分解：λ1, λ2 = eigh(J)
③ λ2/λ1 < ε_c を確認
④ 条件を満たす → DSSCで解ける
   条件を満たさない → J_{ij}を変更

設計コスト：O(N³)（1回だけ）
= DSSCの実行より圧倒的に安い
```

### 3.4 現在解けることが確認された問題クラス

| 問題クラス | 条件 | 実証 |
|-----------|------|------|
| 2複合体識別（N=4） | λ2/λ1=0.667 | L35 ✓ |
| 3複合体識別（N=6） | λ2/λ1=0.571 | L36 ✓ |
| 4複合体識別（N=8） | λ2/λ1=0.500 | L37_A ✓ |
| タンパク質複合体 | 非縮退・符号条件 | L35 ✓ |
| 隠れた構造の検出 | λ2/λ1 < ε_c | L35 S3 ✓ |

### 3.5 現在解けないことが確認された問題クラス

| 問題クラス | 理由 | 実証 |
|-----------|------|------|
| 縮退した境界 | 物理が区別できない | L35 S2 ✓ |
| MAX-CUT（符号反転） | C_{ij}>0のみ | 原理的制約 |
| 動作範囲超過 | sin²の非線形性 | L34 ✓ |
| 階層構造の第2層 | 影効果（λ2/λ1=0.800） | L37_C / L38 ✓ |

## 第5章：設計原則（要件から導出）
### （第3章の要件を実現するための設計指針）

### 5.1 固有値逆設計のフロー

```
Step 1：問題を定義する
  「何のクラスター構造を見つけたいか」

Step 2：J_{ij}行列を設計する
  複合体内結合：J_intra（強）
  複合体間結合：J_inter（弱）

Step 3：固有値比を確認する
  λ2/λ1 = eigh(J)[1] / eigh(J)[0]
  計算コスト：O(N³)、1回だけ

Step 4：要件4を確認する
  λ2/λ1 < ε_c → 進む
  λ2/λ1 ≥ ε_c → Step 2に戻る

Step 5：DSSCを実行する
  s* = 0.890で読み出し
```

### 5.2 配置設計の禁止パターン

```
禁止①：階層配置（L37_C型）
  λ2/λ1が構造的に0.9以上になる
  = J_layer2をいくら変えても解けない

禁止②：交互パターン（L37_B型）
  Nが増えると精度が下がる
  最適なNが存在する

推奨：4複合体配置（ScenarioA型）
  λ2/λ1が(N-2)×J_interで制御できる
  設計が単純で予測可能
```

### 5.3 Nの選択ガイドライン

```
margin ≈ 0.38 × (1-λ2/λ1)^2

同じJ_interでもNで変化：
  小さいN：λ2/λ1が大きい（不利）
  大きいN：λ2/λ1が小さい（有利）

ただし：
  N↑ → 計算コスト ↑（dim=2^N）
  N=8が現実的な上限（PCスペック依存）

推奨：問題の複合体数×2がN
  4複合体を識別したい → N=8
  3複合体を識別したい → N=6
```

### 「なぜ試みたか・何が起きたか・なぜ解けないか」

DSSCの計算クラスは「成功」だけでなく「失敗」によって定義された。

### C.1 禁止①：階層配置（L37_C型）

**目的（何の問題を解こうとしたか）：**
```
タンパク質スーパー複合体の階層的識別問題

具体的な問題設定：
  8つのタンパク質 A〜H が存在する
  既知の相互作用強度：
    二量体結合（第1層）：
      {A,B},{C,D},{E,F},{G,H} → J=1.0（強い直接結合）
    四量体結合（第2層）：
      {A,B,C,D},{E,F,G,H}内  → J=0.4（中程度の間接結合）
    スーパー複合体間（第3層）：
      全ペア               → J=0.05（弱い背景結合）

  「全ての相互作用強度をJ_{ij}に渡したとき
   3層全ての複合体構造を自動識別できるか？」
```

**目標（何を達成しようとしたか）：**
```
階層構造をそのままJ_{ij}に埋め込むことで
DSSCが全3層を自動識別できることを示す

成功すれば：
  「問題の構造をそのまま渡すだけでいい」
  という最も単純な設計原則が確立する
  = 一切の前処理なしにDSSCが使える
```

**試みた配置（L37_C）：**
```
第1層（J=1.0）：{A,B},{C,D},{E,F},{G,H}
第2層（J=0.4）：スーパー複合体内
第3層（J=0.05）：スーパー複合体間

「問題の構造をそのまま物理に渡す」
= 最も直感的な設計
```

**結果（L37_C / L38 Phase1）：**
```
第1層：margin=+0.1247 ✓（検出成功）
第2層：margin=-0.0377 ✗（検出失敗）
第3層：margin=-0.0377 ✗（検出失敗）

L38 Phase1（全sスキャン）でも：
  動作範囲（s≦0.970）内で
  第2層のmarginは最大でも-0.064（負のまま）
```

**なぜ解けないか（物理的説明）：**
```
根本原因：固有値比 λ2/λ1 = 0.800

J_{ij}の設計ではJ_layer2=0.4だが
J行列の固有値分解をすると：
  λ1=2.00, λ2=1.60
  λ2/λ1 = 0.800

= J値とは無関係に
  「行列の固有値構造」が問題

なぜ0.800になるか：
  {A,B,C,D}という4スピンのブロックは
  第1層と第2層が同じスピンを共有している
  → |v1>と|v2>が空間的に重なる
  → 干渉項が大きくなる
  → どのsでも第2層が第1層の「影」に隠れる

いくらJ_layer2を変えても：
  λ2/λ1 ≈ 0.93〜0.96（常に大きい）
  = 階層配置はDSSCの
    固有値構造と相性が悪い

教訓：
  「問題の構造をそのまま渡す」は
  DSSCの物理と合っていない

  DSSCに合う配置は：
  固有ベクトルが空間的に分離した配置
  = 4複合体配置（ScenarioA型）
```

### C.2 禁止②：交互パターン（L37_B型）

**目的（何の問題を解こうとしたか）：**
```
空間的に交互に配置されたタンパク質複合体の識別問題

具体的な問題設定：
  細胞膜上に交互に配列した受容体タンパク質の
  2種類の複合体を識別する

  複合体1（奇数番目受容体）：{A,C,E,G}
  複合体2（偶数番目受容体）：{B,D,F,H}

  物理的配置：A-B-C-D-E-F-G-H（交互に並んでいる）
  複合体内J = 0.17（同複合体の受容体間結合）
  複合体間J = 0.15（異複合体の受容体間結合）

  「隣接せず交互に混在する複合体構造を
   DSSCが正しく識別できるか？」
```

**目標（何を達成しようとしたか）：**
```
4複合体配置（ScenarioA型）と同等の識別精度を
交互パターンで達成することを示す

成功すれば：
  物理的な配置に依らず
  「結合強度の差」だけで識別できる
  = DSSCの汎用性が大きく広がる
```

**試みた配置（L37_B）：**
```
複合体1：{A,C,E,G}（偶数インデックス）
複合体2：{B,D,F,H}（奇数インデックス）

複合体内J（A-C, A-E, A-G, ...）= 0.17
複合体間J（A-B, A-D, ...）      = 0.15

= 構造的に「交互に混在」する配置
```

**結果（L37_B）：**
```
N=4：margin=+0.0418 ✓
N=6：margin=+0.0835 ✓（最大！）
N=8：margin=+0.0215 ✓（減少）

検出は成功したが：
  N=6でピーク後にN=8で急減少
  = N^1.15スケーリングが成立しない
```

**なぜNが増えると悪化するか（物理的説明）：**
```
交互パターンの幾何学的問題：

N=8で複合体内ペア数：
  C(4,2)=6組（{A,C},{A,E},{A,G},{C,E},...）

N=8で複合体間ペア数：
  4×4=16組（{A,B},{A,D},{A,F},{A,H},...）

= 複合体間ペアが複合体内ペアより多い！

Nが増えるほど：
  「識別したい信号」（複合体内）の割合が減少
  「ノイズ」（複合体間）の割合が増加

  N=6でバランスが最良
  N=8では複合体間が多すぎる

さらに：
  {A,C,E,G}という交互配置では
  J行列がブロック対角でなく
  固有ベクトルが複雑に混在する
  → 空間的重なりが4複合体配置より大きい
  → 干渉が増える

教訓：
  「交互パターンは最適Nが存在する」
  N=6が実用的な最適値

  4複合体配置と違い
  「大きいほど良い」が成立しない
  = 問題の幾何学がスケーリングを決める
```
